# Updating Dashboard Backends

This guide explains how to add or modify backend applications on the Developer Dashboard without changing the source code.

## 1. Modify the Configuration
The backend list is managed centrally in the root `values.yaml` file.

**File**: `/home/berrybytes/Desktop/codeInspector/codeInspector/values.yaml`

Add a new item to the `backends` list under the `z1sandbox-website` section:

```yaml
z1sandbox-website:
  # ... other values ...
  backends:
    - id: "MY_NEW_BACKEND"
      name: "Alpha Laboratory"
      description: "Custom description for your new backend service."
      baseUrl: "https://your-api-endpoint.ngrok-free.dev/backend/alpha"
      documentationUrl: "https://your-api-endpoint.ngrok-free.dev/backend/alpha/docs"
      icon: "terminal" # Options: terminal, box, code
      color: "indigo"  # Options: indigo, emerald
```

## 2. Apply Changes to the Cluster
After saving the `values.yaml` file, run the following command to sync the configuration with Kubernetes:

```bash
helm upgrade --install codeinspector ./codeInspector \
  -f ./codeInspector/values.yaml \
  -n default
```

## 3. Refresh the Dashboard
To ensure the website picks up the new configuration, restart the deployment:

```bash
kubectl rollout restart deployment z1sandbox-website -n opensandbox-system
```

---
**Note**: The dashboard fetches its configuration from `/config/backends.json` at runtime. This JSON is dynamically generated by the Helm chart from the values you provided in step 1.
